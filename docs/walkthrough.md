# Walkthrough: Golden C++ PID Precision Landing

The Precision Landing Drone Simulation has been successfully transitioned to a robust C++ PID controller with a verified `make demo-cpp` execution flow. Here is a summary of the achievements and the steps taken:

## 1. Golden C++ PID Implementation
- The C++ node `precision_landing_control_cpp` was developed to subscribe to ArUco marker `/aruco/center_error` and publish target velocities to `/precision_landing/cmd_vel_cpp`.
- The node incorporates robust control logic: deadbands to prevent micro-oscillations, integral windup clamps, finite output saturation, and target-loss stale observation timeouts.

## 2. Shadow Mode Validation
- A shadow mode logger was implemented to passively receive velocity commands generated by both the Python baseline controller and the C++ node under the same runtime conditions.
- **Result:** The C++ controller exactly mirrored the sign outputs of the Python node, with a P95 magnitude discrepancy of `<0.001 m/s`.

## 3. Runtime Repeatability & Fixes
- Addressed flakiness in the `mission_commander.py` node that would occasionally cause a `NO_SETPOINT_SET` rejection when transitioning to `Offboard` mode by implementing a 3-setpoint burst mechanism prior to PX4 `start()`.
- Addressed Docker zombie processes from ungraceful shutdown exceptions that blocked port `14540`.
- **Result:** Executed three consecutive foreground runs of `make demo-cpp`. The system landed successfully in **3/3 runs**, validating 100% repeatability with `0` occurrences of NaN, Inf, or stale state inputs.
- [View Report C3](evidence/REPORT_C3.md) for detailed metrics.

## 4. Visual Polish (Gazebo Environment)
- Following the Repeatability PASS, the Gazebo environment (`inspection.sdf`) was enhanced with Visual Polish.
- We added **6 primitive static visual entities** scattered around the ArUco marker landing pad (a red box, a green box, a blue cylinder, a yellow cylinder, an orange sphere, and a purple box) to emulate a realistic warehouse inspection or factory floor, which also gives the downward-facing camera richer visual features to simulate real-world conditions.

The system is now stable, and you can safely view the environment by running:
```bash
make demo-cpp
```
